# AI 网关

不直接调用模型，而是通过 AI 网关统一接入。

核心功能：
- **多模型统一接入**
	对外开放同一套接口，内部转换为具体的模型接口协议。
	可以：
	- 随时切换模型
	- 做A/B测试
	- 避免厂商锁定
+ **成本控制**（非常关键）
	+ 统计每个用户/每个功能的消耗
	+ 设置额度限制
	+ 自动选择更便宜的模型
+ **请求治理**（稳定性保障）
	当AI API 出现超时、限流、报错后：
	+ 重试机制
	+ 降级策略
	+ 负载均衡
+ **Prompt 管理**（企业级刚需）
	+ 版本管理
		Prompt 也需要不断优化，但是可能优化后的版本效果变差，需要有回滚机制。
		给每一版提示词都加上版本号。
	+ 模版化
		把 Prompt 从“死字符串”变成“动态模板”；
		比如：
		```
		你是一个专业的食谱推荐助手。  
  
		用户需求：  
		{{user_input}}  
		  
		用户偏好：  
		- 口味：{{taste}}  
		- 忌口：{{diet_restrictions}}  
		  
		请推荐 3 道菜，并说明理由。
		```
	+ 在线修改
+ **安全与合规**
	- 敏感信息脱敏（手机号、身份证）
	- 内容审核（防止违规输出）
	- 日志审计（谁问了什么）
+ **日志 & 可观测性**
	+ 记录每次请求的 prompt / response
	+ token使用量统计
	+ 延迟
	+ 成功率
+ **工具调用 / Agent 编排**（进阶能力）
	+ function calling 管理
	- 工具路由（搜索、数据库、爬虫）
	- Agent orchestration（多步骤任务）
