**内置的 14 个技能**：

- **brainstorming**  
    在开始任何开发前进行头脑风暴，明确需求和设计思路。
- **systematic-debugging**  
    系统化调试流程，在修复前找出根本原因。
- **test-driven-development**  
    将 TDD（红‑绿‑重构）作为实现特性或修复 bug 的主流程。
- **writing-plans**  
    把设计和需求拆解成具体的可执行小任务。
- **executing-plans**  
    按实现计划逐步执行编码任务，并设置检查点。
- **requesting-code-review**  
    在实现后触发请求代码审查，提前发现问题。
- **subagent-driven-development**  
    使用子代理并行执行任务，提高自动化开发效率。
- **using-superpowers**  
    初始技能，用于设置技能系统并引导后续流程。
- **verification-before-completion**  
    在声称工作完成前执行验证，确保结果正确。
- **writing-skills**  
    编写/修改 Superpowers 新技能的能力，用于扩展技能库。
- **using-git-worktrees**  
    使用 Git 工作树来隔离不同任务，提高工作流结构性。
- **receiving-code-review**  
    处理来自审查者的反馈并据此调整代码。
- **dispatching-parallel-agents**  
    并行分发多个子代理进行不同任务，提高效率。
- **finishing-a-development-branch**  
    在任务完成并所有测试通过后，决定如何合并/清理分支。
