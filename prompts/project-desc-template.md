# 项目描述模版

Vibe Coding 前首先需要描述清楚项目的一些信息：

- 项目目标
- 面向用户
- 组织架构
	- 功能模块划分
	- 模块功能需求
	- 模块业务流程
	- 模块间交互
- UI 界面（按模块分别设计）
	- 页面内容
		- 内容分段、文案等
	- 内容结构
	- 风格样式
	- 技术方案
- 后端业务（按模块分别设计）
	- 业务流程
	- 技术方案
	- 数据表设计
- 技术栈
- 部署架构
	- 服务器配置选型
- CICD流程
	- CICD方案
	- 版本兼容考虑
- 软件工程设计
	- 质量保证
		- 代码组件化模块化（可复用、可维护）
		- 编码规范
		- 引入测试（可测试性）
		- 文档
	- 安全性
		- 数据安全（零容忍丢失）
			- 数据备份与恢复
		- 访问安全
			- 白名单
			- API 限流器
			- 权限系统
	- 可靠性
		- 分布式集群（避免单点故障、高可用、容灾）
		- 接口可靠性
			- 重试/熔断/降级
			- 部分业务幂等
	- 可扩展性
		- 可随着业务量方便拓展
	- 性能优化
		- 加入接口性能告警
		- 后续按需再优化
			- 缓存
			- 索引
			- 异步
			- 请求聚合
			- 节点横向拓展
	- 可运维
		- 环境变量配置
		- 支持灰度发布与回滚
		- 健康检查
		- 日志和监控（可监控追踪）
		- 告警机制
