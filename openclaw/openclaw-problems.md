# OpenClaw 的问题

## 存在的问题

- **安全性**

	│  如果你对安全加固和访问控制不太熟悉，请不要运行 OpenClaw。                         │
	│  在启用工具或将其暴露到互联网之前，请寻求有经验的人帮助。                           │
	│                                                                                    │
	│  推荐的安全基线：                                                                  │
	│  - 配对/白名单 + 提及限制。                                                        │
	│  - 多用户/共享收件箱：拆分信任边界（使用独立网关/凭证，最好使用独立的操作系统      │
	│    用户或主机）。                                                                  │
	│  - 沙箱 + 最小权限原则的工具。                                                     │
	│  - 共享收件箱：隔离私聊会话（`session.dmScope: per-channel-peer`），并保持工具     │
	│    访问权限最小化。                                                                │
	│  - 将机密信息存放在代理无法访问的文件系统中。                                       │
	│  - 对于任何启用了工具或接收不受信任消息的机器人，请使用最强可用的模型。             │
	│                                                                                    │
	│  定期运行以下命令：                                                                │
	│  openclaw security audit --deep    # 深度安全审计                                  │
	│  openclaw security audit --fix     # 自动修复安全问题                              │
	│                                                                                    │
	│  必须阅读：https://docs.openclaw.ai/gateway/security                               │
