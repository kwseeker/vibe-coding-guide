# OpenClaw 训练

- 如何训练 OpenClaw 完成复杂任务？

以“订外卖”为例。

## 规则制定

- 任务描述
- 任务目标
- 任务约束
- 任务工具
- 反馈机制

**策略**：
- 多 Agent 角色
- 身份信息（IDENTITY.md）、原则(SOUL.md)
	- `IDENTITY.md` 定义 Agent 角色掌握的“资源”和“背景”
		比如：
		- 基本属性： 名字（如 "Claw-Foodie"）、版本、核心职能（如“外卖辅助专家”）。
		- 个人偏好 (Preferences)： * 口味： 不吃香菜、喜欢重辣、对花生过敏。
		- 品牌偏好： 优先选美团、咖啡只喝瑞幸或库迪。
		- 常用资产 (Assets)： * 配送地址： 公司地址、家里地址。
		- 联系方式： 订餐手机号。
		- 权限边界： * 可以访问的 App 列表。
		- 单次消费上限（例如：单笔订单不得超过 100 元）。
		- 历史记忆摘要： 上次订餐的评价记录（例如：上次点的某家店太咸了，标记为“避雷”）。
	- `SOUL.md` 定义了 Agent 在执行任务时如何“思考”和“抉择”，特别是在遇到模糊指令时。
		- 核心指令 (Core Mandate)： * “始终以用户的财务安全为第一优先级。”
		- “在任何涉及金钱支出的操作前，必须获取人类的二次确认。”
		- 沟通风格 (Voice & Tone)： * 是简洁高效型（只报结果），还是体贴建议型（会对比三家店的优惠）。
		- 冲突解决逻辑： * “如果预算与评分产生冲突，优先保证评分在 4.5 以上，即便超出预算 10%。”
		- “如果用户要求的餐厅关门了，自动寻找同品类销量最高的替代品。”
		- 反思与自纠错机制： * “在操作失败两次后，停止尝试并向用户发送详细的错误截图，而不是反复重试导致账号被封。”
		- 伦理与合规： * 不执行任何刷单、虚假评价等违规操作。
- 强化学习微调
- 插件与工具、技能调用
