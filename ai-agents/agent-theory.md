
# AI Agent 工作原理

Agent 从实现角度看本质是对 LLM 有限上下文空间的精细管理。

参考：
+ [AI Agent 工作原理](https://www.runoob.com/ai-agent/ai-agent-working-principle.html)

关键问题：

- **LLM 与 Agent 的关系？**
	LLM 作为 Agent 的决策中心和推理引擎，Agent 可以通过 Tools、记忆 获取实时信息或执行具体操作。

- **LLM 既然可以通过 Agent 拓展数据和功能，为何还需要微调？**
	Agent 解决“不会查、不会用工具”的问题，相当于给人提供更多资源和权限；
	微调解决“不会想、不会表达、不会稳定执行”的问题，优化模型“思考方式 & 行为模式”，相当于提升人的智力和行为方式。

- **什么时候需要微调？**
	当“改 prompt / 用 RAG / 上 Agent”都无法让结果稳定、可控、成本合理时，就该微调。
	不会做事（不稳定）→ 微调；不知道信息 → RAG；不会调用工具 → Agent。
	比如可以将稳定不变的知识（常识、基本公式、基本原理等，不包括用户数据等）能力微调到模型里面，从而降低 Agent 需要填写的上下文。

- **Agent 的组件架构**
	+ **LLM**
	+ **工作流引擎**
	+ **工具**
		经历了 functions_call -> MCP -> SKills 的迭代；
		MCP 并没有被 Skills 取代，两者是互补的关系，MCP 实现通过通道和协议连接工具（主要是API），Skills 则实现可动态加载的可解决某个具体领域问题的提示词。
	+ **记忆**
		分为短期记忆和长期记忆。
		- 短期记忆
			上下文（Context）
		- 长期记忆
			外部存储（文件 / 向量库 / 工具）。知识库 RAG 属于向量库的实现形式。
			这里的工具指将一些没必要常驻内存的提示词封装为 Skills、MCP。

- **Agent 的工作流程**
	主要就是工作流引擎的工作流程，现在多为 ReAct 循环。
	**思考 -> 行动 -> 观察 -> 记忆 -> ...（多轮循环）... -> 得出结论** 。
	工作过程中会通过工具感知环境、通过记忆存储历史和获取的知识以及提取历史和知识参与决策。
