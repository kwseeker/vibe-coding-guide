# 爬虫设计

**AI 推荐的爬虫实践架构**：

方案A：轻量级快速方案

```
发现层 (Tavily) → 爬取+解析层 (Crawl4AI/Firecrawl 二选一) → 存储层
```

方案B：企业级完整方案

```
发现层 (Tavily/Google Search API)
       ↓
调度层 (ScrapyRT/定制调度器)
       ↓
爬取层 (Scrapy/Playwright/Crawl4AI混合)
       ↓
解析层 (BeautifulSoup/LLM提取)
       ↓
清洗层 (pandas/自定义清洗)
       ↓
存储层 (PostgreSQL/MongoDB/OSS)
```

方案C：AI增强方案

```
发现层 (Tavily + LLM筛选)
       ↓
爬取层 (Crawl4AI - 利用其LLM能力)
       ↓
智能解析 (基于LLM的内容提取)
       ↓
知识入库 (向量数据库 + 传统DB)
```

**工作流程**

1. 判断爬取需求是否包含固定的网站网址
    是，则仅爬取这个网站的内容；
	否，则使用 Tavily 搜索和需求相关的网站，并记录网站地址到 `searched-websites.json` 文件。
	> 存储网站地址是为了后续复用和避免重复爬取，存储方式是否需要做**检索优化**？
2. 使用 FireCrawl 等工具深度爬取上面的固定网址或网址列表
	提供**目标网页内容的数据特征**用于页面匹配过滤；
	提供**自定义数据结构**（JSON Schema）用于从上一步匹配的目标网页提取需要的数据（数据清洗）并结构化；
	提供**存储目标位置和格式**，用于存储上一步结构化的数据。
	> **需要理解的原理与注意的限制**：
	> FireCrawl 等工具**深度爬取（全站爬取）原理**：先去寻找网站地图（sitemap.xml）, 如果没有则从入口网页递归点击页面上的链接，直到遍历整个网站。
	> **爬取深度控制**：一般都支持递归爬取深度。
	> **动态内容获取**：现代的网站多是SPA，FireCrawl 等爬虫工具可以像浏览器一样运行JS请求动态内容。
	> **Markdown 预处理**: 它直接把网页 HTML 转换成精简的 Markdown 格式。这对于大模型非常友好，因为它去除了广告、导航栏等“噪音”，节省了 Token。
	> **绕过反爬虫**: 它内置了代理管理和验证码识别，能处理大部分网站的封禁策略。
	> **结构化输出**: 除了 Markdown，它还能直接把网页内容提取成你定义的 JSON 格式（比如：直接提取所有产品的名称和价格）。
	> **限制**：
	> - Robots.txt: 默认会遵守网站的爬取协议。
	> - 身份验证: 如果网页在登录墙后面，通常需要额外配置 Cookie。
	> - 频率限制: 爬得太快太深，对方服务器可能会暂时封锁你的 IP。
3. 数据导入
	提供后端接口或数据库地址，用于将上一步爬取的数据自动导入到后台。
